## Mode Decision

**Advisory mode** (default): Assess whether a website is justified given current surface coverage. Produce a recommendation and sketch. Do not produce a full brief.

Trigger for active mode: the user says they want a website, requests a site brief, or explicitly accepts an advisory-mode recommendation to build one.

**Active mode**: Produce a complete, build-ready website/content brief. Include site job, section map, proof shelf, alignment checklist, and voice notes.

## Proof Shelf

On-site: summaries and highlights that position the user for the target audience.
Off-site: depth links to GitHub repos, case studies, or published work.

Default: summaries on-site, depth off-site. Do not recommend embedding full GitHub repos or resume PDFs directly on the site.

## Builder Boundary

The brand strategy workflow stops at a build-ready handoff. It does NOT produce:
- HTML, CSS, JavaScript, or any implementation code
- Hosting recommendations (Vercel, Netlify, Cloudflare, etc.)
- Domain name suggestions or DNS setup
- Build tool configuration

When the user is ready to build, say: "The builder workflow (Phase 8) handles local site creation when you are ready."

## Alignment Checklist

Check all four surfaces for coherence:
1. Website narrative direction vs LinkedIn headline and about
2. Highlighted GitHub repos vs site proof claims
3. Resume career narrative vs website positioning
4. Off-site proof links — are they live and current?

Flag misalignments explicitly. Do not paper over contradictions.

## Confidence Calibration

- **high**: LinkedIn + GitHub + resume all present, fresh evidence, both Phase 3/4 context available, anti-voice passed
- **medium**: Two of three surfaces, some stale evidence, or missing one Phase context
- **low**: One surface, stale evidence, or both Phase contexts absent

## Advisory Posture

The website brief is for user review. Do not publish, deploy, or implement anything.
