# Research Skill

Research a technical topic and produce structured findings.

## Process

1. **Clarify scope**: Understand what the user wants to know and why
2. **Gather information**: Read relevant code, docs, examples in the project
3. **Evaluate options**: Compare alternatives with pros/cons/tradeoffs
4. **Produce findings**: Structured output with recommendations

## Output Format

### Question
Restate the research question clearly.

### Findings
Key discoveries organized by topic.

### Options Comparison (if applicable)
| Option | Pros | Cons | Recommendation |
|--------|------|------|----------------|

### Recommendation
Clear recommendation with reasoning.

### Sources
List files, docs, or references consulted.

## Guidelines

- Start by exploring the existing codebase for context
- Check package.json, existing patterns, and conventions before recommending new approaches
- Prefer approaches consistent with the project's existing stack
- Flag assumptions and confidence levels
- Note what you could NOT determine (gaps for follow-up)
